DO — Piège #2 : Tools Mal Documentés ⚠️

❌ Description floue = Mauvais choix d'outil = Échecs silencieux

Le LLM ne peut pas "deviner" ce que fait votre fonction. Il se base uniquement sur le nom et la description.

Exemple : Mauvaise vs Bonne documentation

❌ Mauvais

[KernelFunction] [Description("Get data")] public string GetData(string id)

Trop vague, l'agent ne sait pas quand l'utiliser

✅ Bon

[KernelFunction] [Description("Récupère les détails d'une commande client par son ID. Retourne status, items, total.")] public OrderDetails GetOrder( [Description("ID numérique de la commande")] string orderId)

Clair, précis, le LLM sait exactement quoi faire

Best practice : Testez vos tools isolément avant de les donner à l'agent. Validez qu'ils retournent des données exploitables.
