AGENT — Piège #4 : Sur-Orchestration ⚠️

❌ Trop d'agents = Complexité ingérable + Coûts explosifs

Chaque agent ajouté multiplie les interactions, les points de défaillance, et la difficulté de debug.

Principes de simplicité

Commencer simple

  • 1 agent d'abord
  • Ajouter un 2e seulement si nécessaire
  • Maximum 3-5 agents par workflow

Division responsabilités

  • Un agent = une responsabilité claire
  • Éviter les rôles qui se chevauchent
  • Documenter le périmètre de chaque agent

Règle d'or : Si vous hésitez entre 1 agent polyvalent et 3 agents spécialisés, commencez par 1. Divisez seulement quand la complexité le justifie.

Indicateurs qu'il faut diviser

  • L'agent échoue souvent par manque de contexte spécialisé
  • Les instructions dépassent 500 mots
  • Plusieurs domaines d'expertise incompatibles
